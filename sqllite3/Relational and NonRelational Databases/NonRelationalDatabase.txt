Non-Relational Databases (NoSQL)

Definition:
- A Non-Relational Database does NOT store data in traditional tables.
- It stores data in flexible formats like:
 - Documents
 - Key-value pairs
 - Graphs
 - Wide-column stores
- Schema is usually flexible.


Types of NoSQL Databases:

1.Document Database
  - Stores data as JSON-like documents
  Example:
  {
  "name": "Ali",
  "age": 20,
  "courses": ["Math", "Physics"]
}


2.Key-Value Database
  - Like a dictionary in Python
  - Example:
  "user1" → "Yiğit"


3.Graph Database
- Focused on relationships (nodes & edges)



Features:
- Flexible schema
- High scalability
- Faster for large-scale distributed systems
- Not always fully ACID (depends on database)



Examples:
- MongoDB
- Redis
- Cassandra
- Neo4j



When to use:
- Big data applications
- Real-time analytics
- Social networks
- Systems with changing data structure
