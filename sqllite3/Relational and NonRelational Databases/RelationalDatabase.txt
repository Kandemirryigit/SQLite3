Relational Databases (SQL)

Definition:
- A Relational Database stores data in tables (rows and columns).
- Data is structured and organized using relationships between tables.
- It uses SQL (Structured Query Language).




Structure:
- Data is stored in tables
- Tables have:
  - Rows → records
  - Columns → fields
- Tables can be connected using:
  - Primary Key
  - Foreign Key

Example:

Students Table:
id	name	age
1	Ali	    20
2	Ayşe	21


Orders Table:
id	student_id	course
1	     1	     Math


Here:
- student_id is a Foreign Key
- It connects to students.id
- This is called a relationship.



Features:
- Structured schema (fixed columns)
- Uses SQL
- Strong consistency
- ACID properties:
  - Atomicity
  - Consistency
  - Isolation
  - Durability



Examples:
- MySQL
- PostgreSQL
- Oracle Database
- Microsoft SQL Server




When to use:
- Banking systems
- E-commerce
- Systems with strict structure
- When relationships are important
